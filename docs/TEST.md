# Написание Unit-тестов
## 1. Цели и задачи тестирования
### **Цель:**  
 Проверить корректность математических вычислений для геометрических фигур.  
### **Задачи:**  
Покрыть тестами основные функциональные сценарии  
Проверить обработку граничных случаев и ошибочных входных данных  
## 2. Описание тестируемого продукта
Тестируемый продукт представляет собой набор функций для вычисления площади и периметра четырех геометрических фигур: 

- Круг (файл **`circle.py`**)  
- Прямоугольник (файл **`rectangle.py`**)  
- Квадрат (файл **`square.py`**)  
- Треугольник (файл **`triangle.py`**)  

Каждая фигура имеет две функции: **`area()`** и **`perimeter()`**.
## 3. Область тестирования
### Тестированию подлежат:  
- Функции вычисления площади для всех четырех фигур  
- Функции вычисления периметра для всех четырех фигур  
- Обработка граничных случаев (нулевые значения)  
- Обработка некорректных данных (отрицательные значения)  
## 4. Стратегия тестирования
Используется unit-тестирование с фреймворком unittest.  
Для каждой фигуры создан отдельный тестовый класс с набором тестовых методов.

**Пример:**
```python
import unittest

class RectangleTestCase(unittest.TestCase):
    def test_area_normal(self):
        self.assertEqual(area(10, 5), 50)
        self.assertEqual(area(3, 7), 21)

    def test_area_float(self):
        self.assertEqual(area(2.5, 4), 10.0)
        self.assertEqual(area(3.3, 2.2), 7.26)

if __name__ == '__main__':
    unittest.main()
```
## 5. Критерии приемки
Все тесты проходят успешно (статус **"OK"**)  
Покрыты основные математические вычисления  
Обрабатываются граничные случаи
